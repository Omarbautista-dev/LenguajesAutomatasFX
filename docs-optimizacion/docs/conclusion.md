# Conclusión

El proceso de optimización aplicado al proyecto **LenguajesAutomatasFX** permitió analizar el código desde diferentes perspectivas: calidad, legibilidad, mantenimiento y rendimiento.

El uso de herramientas como **SonarQube for IDE**, **VisualVM** e **IntelliJ IDEA** fue fundamental para detectar problemas que no siempre son visibles durante la ejecución normal del programa.

La optimización no consistió únicamente en hacer que el programa fuera más rápido, sino en mejorar la estructura interna del proyecto. Esto permite que el código sea más fácil de entender, modificar y ampliar en el futuro.

## Conclusión general

Después del análisis, se concluye que aplicar buenas prácticas de programación mejora significativamente la calidad de un proyecto. La eliminación de código duplicado, la simplificación de métodos, el uso de nombres descriptivos y la limpieza de código muerto son acciones importantes para mantener un sistema ordenado y profesional.

## Aprendizaje obtenido

Este trabajo permitió comprender que el desarrollo de software no termina cuando el programa funciona. También es necesario revisar, analizar y mejorar el código para asegurar que sea eficiente, mantenible y comprensible.

## Recomendación

Se recomienda continuar utilizando herramientas de análisis durante el desarrollo del proyecto, no solo al final. Esto permite detectar errores desde etapas tempranas y mantener una mejor calidad en el código.
