# Comparativa antes y después

Esta sección presenta una comparación general entre el estado inicial del código y el resultado después de aplicar optimizaciones.

## Comparativa general

| Aspecto | Antes de optimizar | Después de optimizar |
|---|---|---|
| Legibilidad | Algunas partes eran difíciles de entender | Código más claro y ordenado |
| Código duplicado | Existían fragmentos repetidos | Se redujo la repetición |
| Métodos | Algunos métodos concentraban demasiada lógica | Métodos más pequeños y específicos |
| Variables | Algunos nombres no eran descriptivos | Nombres más claros |
| Código muerto | Existían fragmentos innecesarios | Se eliminaron partes sin uso |
| Mantenimiento | Más difícil de modificar | Más fácil de mantener |
| Rendimiento | Posible uso innecesario de recursos | Mejor uso de recursos |

## Impacto de la optimización

La optimización permitió mejorar la calidad del proyecto sin modificar su funcionamiento principal. El usuario final no percibe cambios negativos, ya que la aplicación mantiene sus funciones, pero internamente el código es más limpio y eficiente.

## Beneficios obtenidos

- Mejor organización del código.
- Reducción de redundancias.
- Mayor facilidad para corregir errores.
- Mejor comprensión del proyecto.
- Preparación del código para futuras mejoras.
