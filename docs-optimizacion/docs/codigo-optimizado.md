# Código optimizado

En esta sección se documentarán las mejoras aplicadas al código del proyecto. Cada optimización debe conservar el mismo resultado funcional del programa, pero con una estructura más clara, eficiente o mantenible.

## Criterios de optimización aplicados

Las mejoras se basaron en los siguientes criterios:

- Simplificación de lógica.
- Eliminación de código duplicado.
- Eliminación de código muerto.
- Mejora en nombres de variables y métodos.
- Reducción de operaciones innecesarias.
- Separación de responsabilidades.
- Mejora en la legibilidad del código.

## Optimización 3: Mejora de nombres de variables

### Código optimizado

```java

int posicionActual;
boolean esOperadorValido;
```

### Explicación

Se reemplazaron nombres genéricos por nombres más descriptivos que explican mejor el propósito de cada variable.

### Beneficio

- Mayor legibilidad.
- Código más comprensible.
- Mejor mantenimiento.
